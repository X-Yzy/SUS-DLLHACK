# SUS-DLLHACK

基于“白加黑”技术的自动化 DLL 劫持检测与代理 DLL 生成 Qt 系统。

## 项目简介

SUS-DLLHACK 是一个《语言课程设计》QT课程作业的本地安全实验工具，主要用于学习 DLL 劫持、白加黑技术、导出函数验证、代理 DLL 自动生成等安全工具原理。

项目通过图形化界面封装扫描、验证、日志分析和代理 DLL 生成流程，减少手工查找白文件、分析导出函数、编写代理 DLL 模板的重复工作，便于在本地封闭环境中进行自动化测试与课程展示。

## 主要功能

- 扫描结果自动进入批量劫持验证队列。
- 只展示实际命中的可用导出函数。
- 生成代理 DLL 页面支持选择验证结果、加载/保存模板、上传 bin，并可选 sgn 混淆生成 `code.data`。
- `code.data` 生成时可选 XOR 加密，密钥可自定义，默认 `0x5A`。
- 内置两套 A() 模板：直接加载执行模板、XOR 解密后执行模板。
- 可从本地 `log` 目录载入历史验证出的导出函数。
- 最终输出目录包含白文件、代理 DLL、生成源码、`code.data` 和编译日志。
- 运行期不再携带完整 LLVM / WinSDK / MinGW，改为约 24.5MB 的极简 MinGW C 编译链。

## 编译说明

本项目提供两种编译方式。

### 方式一：一键编译脚本（需要自行修改bat中的路径）

推荐直接运行项目根目录下的自动编译脚本：

```bat
build_all.bat
```

脚本会自动完成以下工作：

- 配置 CMake 构建目录。
- 使用 Qt、CMake 和 clang-cl 编译主程序。
- 自动查找并打包运行期所需的极简 MinGW 编译链。
- 调用 `windeployqt` 部署 Qt 运行库。
- 生成最终程序：`build\SUS-DLLHACK.exe`。

### 方式二：按照 docs 文档手动编译

如果需要手动配置环境、排查编译问题或用于课程报告说明，可以参考文档中的完整命令：

- [编译运行命令.md](docs/编译运行命令.md)

手动编译适合需要自定义 Qt 路径、CMake 路径、编译器路径或 MinGW 路径的情况。

```text
SUS-DLLHACK/

│   ├── ZeroEye/            # ZeroEye 扫描逻辑
│   └── sgn/                # sgn-lite 混淆逻辑
├── docs/                   # 项目说明、编译运行文档、展示材料
├── CMakeLists.txt          # CMake 构建配置
└── build_all.bat           # Windows 一键编译脚本

```
